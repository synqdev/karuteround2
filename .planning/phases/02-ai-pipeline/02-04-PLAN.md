<task type="auto">
  <name>Task 1: Pipeline orchestration function with retry logic</name>
  <files>src/lib/ai-pipeline.ts</files>
  <action>
Create `src/lib/ai-pipeline.ts`:
- Export `runAIPipeline(audioBlob: Blob, locale: string, onProgress: (step: PipelineStep) => void)` function
- Define `PipelineStep` type: `'transcribing' | 'extracting' | 'summarizing' | 'complete' | 'error'`
- Export `PipelineStep` type and `PipelineResult` type: `{ transcript: string, entries: Entry[], summary: string }`
- Implementation:
  1. Call `onProgress('transcribing')`
  2. POST audio blob as FormData to `/api/ai/transcribe` with locale — wrap in `fetchWithRetry` helper
  3. Parse transcript from response
  4. Call `onProgress('extracting')` (note: extraction and summary happen in parallel, but the step label indicates we are in the "AI processing" phase)
  5. Run extraction and summary in parallel via `Promise.all`:
     - POST `{ transcript, locale }` to `/api/ai/extract`
     - POST `{ transcript, locale }` to `/api/ai/summarize`
  6. Call `onProgress('complete')`
  7. Return `{ transcript, entries, summary }`

Create a private `fetchWithRetry` helper:
- Takes a fetch call (as a function), retries once on failure (per user decision — auto-retry once)
- On first failure: wait 1-2 seconds, retry once
- On second failure: throw the error (caller handles it)
- Use Claude's discretion for retry timing

Handle errors: if any step fails after retry, throw with a descriptive error message. The caller (PipelineContainer) catches and shows error UI.
  </action>
  <verify>Run `npx tsc --noEmit` — no type errors. Verify Promise.all is used for extraction + summary (not sequential). Verify fetchWithRetry retries exactly once.</verify>
  <done>Pipeline function orchestrates transcribe → parallel extract+summarize with progress callbacks and auto-retry. Returns transcript + entries + summary.</done>
</task>

Create `src/components/review/PipelineContainer.tsx`:
- The orchestrating component that manages the full flow: processing → review
- Props: `{ audioBlob: Blob, locale: string, onConfirm: (data: { entries: Entry[], summary: string }) => void, onCancel?: () => void }`
- State: `phase: 'processing' | 'review'`, `currentStep: PipelineStep`, `error: string | null`, `result: PipelineResult | null`
- On mount (useEffect): calls `runAIPipeline(audioBlob, locale, setCurrentStep)`
  - On success: stores result, sets phase to 'review'
  - On error: sets error state (ProcessingModal shows retry button)
- Retry handler: clears error, re-runs `runAIPipeline`
- When phase is 'processing': renders ProcessingModal
- When phase is 'review': renders ReviewScreen with result data, passing onConfirm through
- Does NOT persist anything to Supabase — that is Phase 4's concern. The onConfirm callback is for the parent to handle.
