  <action>
    **Create `src/actions/customers.ts`** — Server Actions for customer CRUD:

    - Add `'use server'` directive at top
    - Define `CustomerFormSchema` with Zod:
      - `name`: string, min 1 ("Name is required"), max 100
      - `furigana`: string, max 100, optional, or empty string
      - `phone`: string, max 20, optional, or empty string
      - `email`: string, email validation, optional, or empty string
    - Define return type `ActionResult` as `{ success: true; id: string } | { success: false; error: string; duplicateWarning?: string }`

    `createCustomer(input)`:
    1. Parse input with CustomerFormSchema
    2. Call `checkDuplicateName(name)` from queries.ts — if match found, include `duplicateWarning: "A customer named X already exists"` in the response but still proceed with creation (per user decision: warn but allow saving)
    3. Get Supabase server client via `createClient()` from `@/lib/supabase/server`
    4. Insert into customers table: `{ name, furigana: furigana || null, phone: phone || null, email: email || null }`
    5. Use `.insert([...]).select().single()` to get the new record back
    6. Call `revalidatePath('/customers')`
    7. Return `{ success: true, id: data.id }` — do NOT call `redirect()` here; let the UI handle navigation after sheet closes (per user decision: sheet closes, list refreshes, toast)
    8. Wrap Supabase call in try/catch, return `{ success: false, error: message }` on failure

    `updateCustomer(id, input)`:
    1. Parse input with CustomerFormSchema
    2. Get Supabase server client
    3. Update customer by id: `.update({ name, furigana, phone, email }).eq('id', id)`
    4. Call `revalidatePath('/customers')` and `revalidatePath('/customers/${id}')`
    5. Return `{ success: true, id }` or `{ success: false, error }`

    **Create `src/lib/customers/queries.ts`** — query helpers:

    `listCustomers({ query, page, pageSize, sortBy, sortOrder })`:
    - Default pageSize: 10 (per user decision)
    - Default sortBy: 'updated_at' (proxy for last visit — closest available column until karute_records exist)
    - Default sortOrder: 'desc' (most recent first per user decision)
    - Build Supabase query: `.select('*', { count: 'exact' })`
    - If query string provided, use `.or()` to search across multiple columns:
      `name.ilike.%${query}%,furigana.ilike.%${query}%,phone.ilike.%${query}%,email.ilike.%${query}%`
      This matches the user decision: "matching name + furigana + contact info (phone/email)"
    - Apply `.order(sortBy, { ascending: sortOrder === 'asc' })`
    - Apply `.range(from, to)` for pagination
    - Return `{ customers, totalCount, totalPages }`

    `getCustomer(id)`:
    - `.select('*').eq('id', id).single()`
    - Return customer or throw if not found

    `checkDuplicateName(name)`:
    - `.select('id, name').ilike('name', name).limit(1)`
    - Return `{ exists: boolean; existingName?: string }`

    **Create `src/lib/customers/utils.ts`** — utility functions:

    `getAvatarColor(customerId: string)`:
    - Deterministic pastel color from customer UUID (per user decision: consistent per-customer, not random on each render)
    - Hash the UUID string to a number, use modulo to pick from a palette of 8-10 pastel colors suitable for dark theme
    - Palette: soft pastels that contrast well on dark backgrounds (e.g., pastel blue, pink, green, yellow, purple, coral, teal, lavender)
    - Return `{ bg: string; text: string }` with Tailwind class strings

    `getInitials(name: string)`:
    - For Latin names: first letter of first two words (e.g., "John Smith" -> "JS")
    - For CJK names: first 1-2 characters (e.g., "田中太郎" -> "田中")
    - Detect CJK by checking Unicode range
    - Return uppercase string
